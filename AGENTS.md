# AGENTS.md

## Repository Shape
- This repo has separate Go modules, not a root `go.work`; run Go commands from `comet-sdk/` or `cometmind/`, not the repository root.
- `comet-sdk/` is the provider-agnostic LLM I/O library. Public entrypoints are `sdk.go` plus `llm.GenerateMessage` and `llm.StreamMessage`; providers live under `provider/anthropic` and `provider/openai`.
- `cometmind/` is the local agent runtime, CLI/TUI, SQLite persistence, and localhost HTTP/SSE API. Its `go.mod` uses `replace github.com/cometline/comet-sdk => ../comet-sdk`, so SDK edits can affect CometMind immediately.
- `cometline/` contains the SvelteKit + Electron desktop shell and is executable; run it with `make dev` from the repository root.

## Commands
- `cd comet-sdk && go test ./...` runs SDK unit/integration tests without live API calls; equivalent Makefile target is `make test`.
- `cd comet-sdk && go test -run TestName ./path/to/package` runs one SDK test; examples: `go test -run TestStream_TextOnly ./provider/anthropic`.
- `cd comet-sdk && go test -tags=live -timeout=60s ./...` runs live provider tests and requires API keys; use `make test-live`, `make test-live-anthropic`, or `make test-live-openai` for the documented variants.
- `cd comet-sdk && go build ./...` or `make build` verifies SDK compilation.
- `cd cometmind && go test ./...` runs CometMind tests.
- `cd cometmind && go test -run TestName ./path/to/package` runs one CometMind test; examples: `go test -run TestPostMessageStreamsSSEAndPersistsUserTurn ./server`.
- `cd cometmind && go build ./...` verifies CometMind compilation.
- There is no repo-wide lint config. `comet-sdk` has `make lint`, which requires `golangci-lint`; `cometmind` has no Makefile target.

## Generated Code And Database
- `cometmind/internal/db/*.sql.go`, `db.go`, and `models.go` are generated by sqlc from `cometmind/internal/db/schema.sql` and `cometmind/internal/db/queries/*.sql`; do not hand-edit generated files.
- After changing CometMind schema or queries, run `cd cometmind && sqlc generate` if sqlc is available.
- CometMind embeds `internal/db/schema.sql` and tracks SQLite migrations with `schemaVersion` plus `alterStatements` in `internal/db/migrate.go`; schema changes for existing users need an incremental migration there, not just a schema edit.
- CometMind uses pure-Go `modernc.org/sqlite` and stores its default database at `~/.cometmind/cometmind.db`.

## Runtime And Environment
- CometMind config is `~/.cometmind/config.toml`; `go run . init` creates config/database and registers the current workspace.
- `COMETMIND_*` environment variables override config fields such as `COMETMIND_BASE_URL`; provider API keys are `ANTHROPIC_API_KEY` or `OPENAI_API_KEY`, falling back to `COMETMIND_API_KEY`.
- `go run . serve --port 7700` binds the local API to `127.0.0.1`; the OpenAPI source of truth is `cometmind/openapi.yaml`.
- `go run . chat "message"`, `go run . tui`, and `go run . session list` all scope sessions to the current directory unless `--workspace` is supplied.
- Built-in CometMind tools are registered in `internal/tools/registry.go`; file tools are workspace-scoped through `internal/tools/sandbox/pathcheck.go` and should not allow path escape.

## Tests And Fixtures
- SDK live tests are behind the `live` build tag and intentionally skipped by normal `go test ./...`.
- SDK live tests use `CUSTOM_API_KEY` first, then provider-specific keys; `CUSTOM_BASE_URL` can override both Anthropic and OpenAI endpoints.
- SDK stream tests rely on checked-in SSE fixtures under each provider's `fixtures/` directory; update fixtures with parser behavior changes.
- Server tests in `cometmind/server` use `httptest` and temporary SQLite databases, so they should not require a running local server.
